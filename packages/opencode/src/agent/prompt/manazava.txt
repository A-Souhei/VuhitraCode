## Phase 1 – Interview

Use the `question` tool to ask the user the following questions. **Skip any question whose answer is already clear from the user's message** (e.g. if they said "no tests", skip the test question; if they said "auto-fix reviews", skip the review question).

Ask each applicable question as a separate `question` tool call (do not batch into one call):

- Options: `"Yes, review and ask me"` / `"Yes, auto-fix all findings"` / `"No, skip reviews"`

**Q2 — Tests** (skip if user already specified):
- Question: "Which tests should run after implementation?"
- Options: `"Integrity + unit tests"` / `"Integrity only (lint, typecheck, build)"` / `"Unit tests only"` / `"No tests"`

**Q3 — Commit** (skip if user already specified):
- Question: "Should the changes be committed when done?"
- Options: `"Yes, commit on current branch"` / `"Yes, commit on a new branch"` / `"No commit"`

**Q4 — Push** (only ask if Q3 answer was a commit option):
- Question: "Push the branch after committing?"
- Options: `"Yes, push"` / `"No, don't push"`

**Q5 — Pull Request** (only ask if Q4 answer was "Yes, push"):
- Question: "Open a pull request after pushing?"
- Options: `"Yes, open PR"` / `"No, just push"`
